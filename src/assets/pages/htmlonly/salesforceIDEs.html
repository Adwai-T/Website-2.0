Salesforce

Salesforce Development with VScode

First Download the salesforce cli and install it.

To check sfdx is installed successfully in terminal use sfdx -v, it will give a version of sfdx and node installed on the system.

If there is an error, try installing sfdx again, or check if it is added to the class path for windows operating system. Also check nodejs is installed on the system and is added to the class path on the operating system.

To update existing version of sfdx use sfdx update.

Download and Install VsCode.

If you use VsCode for other developement, or other project and would like to setup a different Vscode install just for development of Saleforce. You could try Portable version, guide here.

The protable version will considerable declutter the development enviroment and give you a clean vscode version to work with salesforce.

After vscode is installed, and open, install the Salesforce Extension Pack (Expanded) or Salesforce Extension Pack from the vscode extensions tab on the left side bar.

This will install all the tools that salesforce provide for developing and integrating saleforce org with vscode, so data from org can be accessed in vscode and send from vscode to your organization.

After installing the extensions, vscode is ready for development.

Configure Prettier

If some of the plugins like prettier are not working out of the box after intalling, do npm install, this will install the packages from package.json. Also there are many predefined scripts in package.json that might be helpful.

Prettier can also be further configured to support lwc template formatting, by changing its parser from html template to lwc.

There are several ways to configure prettier that can be found on the prettier website here.

The simplest being adding a prettier option to package.json.


"prettier": {
  "overrides": [
    {
      "files": "∗.html",
      "options": {
        "parser": "lwc"
      }
    }
  ]
}

All of these configuration are local and do not need to be and are not commited to remote server.

Installing Java Development Kit (JDK)

Java (JDK) is required for some features to work, particularly for apex support, so make sure JDK is also installed if there are errors with some plugins. Salesforce Guide on Setting up Java with download links.

Once JDK is installed on system, add the home path of JDK in File | Preferences | Settings | search apex | salesforcedx-vscode-apex.java.home.

There are also other setting that can be configured for the plugins that have been installed.

Create a new Project

Before an org can be connected to the project in vscode, a blank project needs to be created.

Open Command pallete from View | Command Pallete or using shortcut F1 or Ctrl+Shift+P or Cmd + Shift + P.

Type in SFDX: Create Project and follow to create a new project.

Connect Organization with project

Once a blank project is created an org can be connected to the project to get classes and other metadata for development and deploy data back to org once developed.

To connect an org, open command pallete and search SFDX: Authorize an Org.

Select the type of org from the list. If using developer org, select production.

Enter a name for that will represent that org in the local project.

Then salesforce cli will open a new tab in the browser and take to the saleforce login page.

Login to the org that need to be connected. After logging in, salesforce will ask for permission. Allow this permission.

Once allowed the local vscode developement enviroment is ready to get and deploy data to that org. Salesforce CLI will remember credential for future authentication and authorization.

Getting Data from organization

Classes and other meta data can be fetched from organization and deployed to the connected organization.

To do that use the Cloud button now available on the left side nav bar. The Cloud button is available when in a saleforce project that has a connected organization.

From the tab, all that can be fetched from the org is seen. Click on the cloud with down arrow button to download it for local development.

Classes, Triggers, Aura, LWC etc fetched from the org is seen in the force-app/main/default folder of the project.

To deploy data back to organization, right click on the class and select SFDX: Deploy this Source to Org.

There are other options to delete from org, retrive changes from org and so on.

Developer Hub

A scratch org is a dedicated, congfigurable, and short-term Salesforce environment that can quickly spin up when starting a new project, a new feature branch or a feature test.

A Developer Hub is the main salesforce or that can be used to create and manage scratch orgs.

After Dev Hub is enabled on an org, it cannot be disabled.

To Enable Dev Hub in an org goto Setup | Developement | Dev Hub.

Now that the org is set to create Scratch Org, connect the org with vs code project.

Create a Scratch Org

In vs code open Command pallet and use SFDX: Create Default Scratch Org. Then the defaults can be accepted or changes can be made as needed.

Creating a scratch org can take some minutes. Once created a success message is displayed.

To Open the default scratch org use SFDX: Open Default Org in command pallet in vscode.

At the bottom left corner of vs code, there is a button(open org), that will open the scratch org, if salseforce plugins are installed..

When code is send to a scratch org it is said to be pushed to the org and not deployed. Deployed is used for the actual org.

To Push code to scratch org use SFDX: Push Source to Default Scratch Org.

Using SFDX in terminal

Using the commands provided by the saleforce extensions in vscode is simpler and also avoids errors. So it is recommended to use them.

sfdx will give helpful commands that can be used.

sfdx commands will give a list of all the commands supported by sfdx cli.

sfdx update to update installed sfdx cli.

sfdx -v or sfdx --version to check the version of sfdx cli

sfdx plugins --core to see the core plugins included with the sfdx installation.

sfdx force:project:create -n "project name" - to create a new project.

sfdx force:auth:web:login -a aliasName -d - this will open the saleforce login page to get authorization in the default browser.

  • -a defines the alias for the org.
  • -d to make it default for this project

sfdx force:org:create -a scratchOrgAlias -d 7 -f config/project-scratch-def.json -s

  • -d in this above case is number of days the scratch org lives and default is 7.
  • -f is the configuration file path for the scratch org.
  • -s use the same username while createing the scratch org as the current hub.
By default the scratch org do not have any data, but if data is needed in the scratch org from hub org, then the configuration(config/project-scratch-def.json) file should have an additional option "hasSampleData" : true, add if not already present.

sfdx force:org:open - To Open org from command line, this will open the default scratch org in default browser.

sfdx force:lightning:component:create -n myFirstWebComponent -d force-app/main/default/lwc --type lwc - To create a new Lightning web component.

  • -n defines the name of lwc folder and its files.
  • -d defines the target directory where the lightning web component should be created. The target directory must be named lwc.
  • --type specifies what is to be created aura or lwc.

force:source:deploy - To deploy to org.

sfdx force:source:push - To push to scratch org salesforce.

sfdx force:source:retrieve - to retrieve the metadata.

Delete LWC

  • sfdx force:source:delete -m LightningComponentBundle:LWCName -u username@company.com - From cli where LWCName is the name of component.
  • SFDX: Delete from Project and Org - In vscode.
  • Delete the component folder and use - sfdx force:source:push --forceoverwrite - Don't recommend using as it may or may not work.
