Salesforce Visualforce Pages

Introduction To Visualforce Pages

Visualforce is a web development framework that enables developers to build sophisticated, custom user interface for both mobile and desktop apps that can be hosted on the Lightning Platform.

Developers create a visualforce page by composing components, Html, and optional styling elements. Visualforce page is similar to a Standard web page by includes powerful features to access, display and update organization's data.

Each page is accessible at a unique URL.

When a page is accessed, the server performs any data processing, renders the page into HTML and return the results to the browser to display.

Visual Force Standard controller is a part of the visaulforce framework that provides automatic data access and modification, standard actions and more.

Use of Visualforce Pages

  • They can be accessed from the App Launcher.
  • Add a Visualforce page to navigation bar.
  • Add a visaul force page as a component in the lightning App Builder.
  • Launch a Visualforce Page as a Quick Action.
  • Display a Visaulforce Page by Overriding Standard buttons or links.
  • Display a Visualforce Page using custom buttons and Links.

Create Visaulforce pages

Developer Console can be used to create Visualforce pages.

In Developer Console File | New | Visualforce Page to create a new page.

To see the how the new page renders, click Preview. The preview window automatically update to show the changes when code is saved. This preview however shows the page without the salesforce styling.

To view how the page would look in the context of salesforce lightning experience go to salesforce home or any lightning experience window, open the developer console for browser and add the following code.

Change pageName in the following code.


$A.get("e.force:navigateToURL").setParams(
    {"url": "/apex/pageName"}).fire();

The javascript code fires the Lightning Experience navigateToURL event, and is equivalent to entering classic /apex/PageName.

To Open an existing visualforce page, in developer console, goto File | Open.

To view all visualforce pages in the org go to Setup | Custom Code | Visualforce Pages.

While creating a page, developer console can suggest possible attributes that the tag supports.

Components of Visaulforce page

<apex:pageBlock> is a structured user interface element that groups related items on a page.

<apex:pageBlockSection> component can be added inside <apex:pageBlock> to give even more structure. When rendered elements can be collapsed by the user to hide all but the section title. <apex:pageBlockSection> is a child tag and cannot be used without <apex:pageBlock>.

Output Components

<apex:details />- Reproductes the entire standard view page for the sObject of concern. All styling for this component is like that in salesforce classic view.

In apex:details the related list for the record are also included, this can be turned off by using <apex:detail relatedList="false"/>

<apex:relatedList> - Use to display lists of records related to current record.

A particular related list for object can be shown by using-

<apex:relatedList list="Contacts"/> or

<apex:relatedList list="Opportunities" pageSize="5"/>

<apex:enhancedList> and <apex:listViews> are other coarse-grained components that you might want to use with or in place of <apex:relatedList>

<apex:outputField> to display individual fields from a record. Ex. <apex:outputField value="{! Account.Name }"/>

Using apex:outputField by themseleves do not have any label, also they lack the formatting that might be expected.

But when they are wrapped in <apex:pageBlock> and <apex:pageBlockSection> the are displayed in a layout and format that is desired.


<apex:page standardController="Opportunity">
    <apex:pageBlock title="Opportunity Details">
        <apex:pageBlockSection>
            <apex:outputField value="{! Opportunity.Name }"/>
            <apex:outputField value="{! Opportunity.Amount }"/>
            <apex:outputField value="{! Opportunity.CloseDate }"/>
            <apex:outputField value="{! Opportunity.Account.Name }"/>
        </apex:pageBlockSection>
    </apex:pageBlock>
</apex:page>

<apex:outputLink> is used with URLFOR() and $Action to provide direct link for that action. They can be used with tables and iterators to provide links for different records or related reords of an object.

{!URLFOR(target, [id], [inputs], [no override])}.

Replace target with the URL or action, s-control, or static resource merge variable, id with an optional reference to the record, and inputs with any optional parameters. The no override argument is also optional and defaults to “false.” It applies to targets for standard Salesforce pages such as $Action.Account.New. Replace no override with “true” when you want to display a standard Salesforce page regardless of whether you have defined an override for it elsewhere.

<apex:outputText> is used to display text values.

<apex:outputText value="{! $ObjectType.Contact.Fields.FieldName.Label }"/> can be used to display the name of field. This might be useful if the app is being used by multilingual clients, as the field name value will be handled by salesforce and provide translations.


<!--Consider that contact is a variable that represents a record of Contact-->
<apex:outputLink
    value="{! URLFOR($Action.Contact.Delete, contact.Id) }">
    Del
</apex:outputLink>

Display a Table

Use <apex:pageBlockTable> to add a table of data to a page.

<apex:pageBlockTable> is an iteration component that generates a table of data, complete with platform styling.

For example, {!Account.contacts} is an expression that evaluates to a list of contacts for an account.

Use this expression with an iteration component to create a list or table with details of these related contacts.


<apex:pageBlock title="Contacts">
   <apex:pageBlockTable value="{!Account.contacts}" var="contact">
      <apex:column value="{!contact.Name}"/>
      <apex:column value="{!contact.Title}"/>
      <apex:column value="{!contact.Phone}"/>
   </apex:pageBlockTable>
</apex:pageBlock>
and are iteration components for creating tables and lists without styling. And is an iteration component that you can use to generate any arbitrary markup for a collection of records.

Varibles in Visualforce

Dynamic data is accessible in markup through the use of global variables, calculations and properties made avaiables by the page's controller.

A visalforce expression is any set of literal values, variables, subexpressions or operators that can be resolved to a single value. Method calls aren't allowd in expressions.

Expression syntax in visualforce is {! expression} which is also continued in Aura framework. Expressions are case-insensitive and pages are ignored.

Anything inside the {! } delimiter is evaluated and dynamically replaced when the page is rendered or when the value is used. Whitespace is ignored.

The resulting value can be primitive, boolean, sObject, a controller method such as a action method or other results.

Global Variables

Global variables can be accessed as {! $GlobalVariableName.FieldName }.

Visual for provides information about logged-in user in a global variables named $User and can be used as {! $User.FirstName }. This is similar to using UserInfo in Apex code.

There are many global variables that can be used with visualforce pages. A full list can be found here.


<apex:page >
    <apex:pageBlock title="User Info">
        <apex:pageBlockSection title="User Name">
            First Name - {! $User.FirstName }
            <br/>
            Last Name - {! $User.LastName } 
            <br/>
            UserName - {! $User.UserName }
        </apex:pageBlockSection>
        <apex:pageBlockSection title="Is Active">
            Is user Active - {! $User.isActive } 
        </apex:pageBlockSection>
    </apex:pageBlock>
</apex:page>

Formula Expressions

Formulas in expression are evaluated to get a value that will be used.

Examples of some formula expressions.

{! $User.FirstName & ' ' & $User.LastName } - here & is used to concatenate strings.

{! TODAY() + 7 } - Gives a date 7 days after the date that the page is being accessed.

{! Max(1, 2, 3) } - Gives max of provided number.

{! CONTAINS('salesforce', 'force')} - return boolean value by checking first string for second string.

It can be considered any fromula that can be used in the formula field in objects can be used here in visualforce pages.

Conditional expressions just like in formula fields can be used in expressions.


<p>{! IF( CONTAINS('salesforce.com','force.com'),
     'Yep', 'Nope') }</p>
<p>{! IF( DAY(TODAY()) < 15,
     'Before the 15th', 'The 15th or after') }</p>

Visualforce Standard Controller

Visualforce uses the traditional model–view–controller (MVC) paradigm, and includes sophisticated built-in controllers to handle standard actions and data access, providing simple and tight integration with the Lightning Platform database.

Most Standard and all custom objects have standard controllers that can be used to interact with the data associated with the object. These controllers can be extended to add new functionality, or custom controllers can be created from scratch.

The standard controller in addition to fetching data also provides a set of standard actions, such as create, edit, save, and delete, that can be added to the pages using standard user interface elements such as buttons and links.

Use Standard Controller to Fetch data for Single sObject

Go the visualforce preview and add the record id to the url like https://resourceful-moose-dne36d-dev-ed--c.visualforce.com/apex/DisplayUserInfo?core.apexpages.request.devconsole=1&id=001D000000JRBes.

The above url now has two parameters. core.apexpages.request.devconsole=1 which already existed and id that we added. Anything after ? is a url parameter and are separated by &.

To preview the page in the context of salesforce lightning experience with the id parameter use-


$A.get("e.force:navigateToURL").setParams(
    {"url": "/apex/pageName?&id=00141000004jkU0AAI"}).fire();

Now that the record id is in the url the standard controller can fetch the data for that record.

Lets take that the record id is for an account. A standard conroller for account can then fetch data for the record and store it in the controller for it to be used in the page.


<!-- URL should have valid record id as parameter to display Account info-->
<apex:page standardController="Account">
    <apex:pageBlock title="Account Summary">
        <apex:pageBlockSection>
            Name: {! Account.Name } <br/>
            Phone: {! Account.Phone } <br/>
            Industry: {! Account.Industry } <br/>
            Revenue: {! Account.AnnualRevenue } <br/>
        </apex:pageBlockSection>
    </apex:pageBlock>
</apex:page>

Input Data Using Forms

Use <apex:form> and <apex:inputField> to create a page to edit data. Combine <apex:commandButton> with the save action built into the standard controller to create a new record, or save changes to an existing one.

When using the standard contorller, apex:form and apex:commandButtons, visualforce does most of the work required. Like retrieve data, display relevant data, validate new values, save valid changes, and provide appropriate messages to the person submitting the changes.

If data needs to be send back to salesforce, most of the times this will be done inside an <apex:form>.

Place form elements within <apex:pageBlock> and <apex:pageBlockSection> tags to organize and group them, and to have the form adopt the platform visual style.

The <apex:inputField> component renders the appropriate input widget based on the standard or custom fields type. 

<apex:page standardController="Contact">
    <apex:form>
        <apex:pageBlock title="Contact Details">
            <apex:pageBlockSection>
                <apex:inputField value="{! Contact.FirstName}"/>
                <apex:inputField value="{! Contact.LastName}"/>
                <apex:inputField value="{! Contact.Email}"/>
                <apex:commandButton action="{! save }" value="Save"/>
            </apex:pageBlockSection>
        </apex:pageBlock>
    </apex:form>
</apex:page>

Display Form Errors and Messages

Use <apex:pageMessages> to display any form handling errors or messages.

When using the standard controller, it collects all the errors in the form. <apex:pageMessages> then tells the controller where to display these messages.

<apex:inputField> displays errors for its own field, where as <apex:pageMessages> are used to display errors for the whole form.

Edit Related Records

<apex:outputLink> with standard controller can be used to make it easy for users to edit related information by providing links to related records. <apex:pageBlockTable> can be used to iterate over all the related records and given provide relevant links to the outputLink tags.

Example -


<apex:pageBlock title="Contacts">
    <apex:pageBlockTable value="{!Account.contacts}" var="contact">
        <apex:column>
            <apex:outputLink
                value="{! URLFOR($Action.Contact.Edit, contact.Id) }">
                Edit
            </apex:outputLink>
            &nbsp;
            <apex:outputLink
                value="{! URLFOR($Action.Contact.Delete, contact.Id) }">
                Del
            </apex:outputLink>
        </apex:column>
        <apex:column value="{!contact.Name}"/>
        <apex:column value="{!contact.Title}"/>
        <apex:column value="{!contact.Phone}"/>
    </apex:pageBlockTable>
</apex:pageBlock>

Standard List Controllers

Standard List controller can be used to create Visualforce pages that can display or act on a set of records.

Standard List conrollers queries for records of a specific object and makes the records available in a collection variable. It also filters and paginates through the result.

As instead of a single record in this case there is a collection of records an iteration component is used to display the results.

First add standardController attribute and then add recordSetVar attribute to the same page to make it a Standard List controller.

Following is an example of Contact.


<apex:page standardController="Contact" recordSetVar="contacts">
    <apex:pageBlock title="Contacts List">
        <!-- Contacts List -->
        <apex:pageBlockTable value="{! contacts }" var="ct">
            <apex:column value="{! ct.FirstName }"/>
            <apex:column value="{! ct.LastName }"/>
            <apex:column value="{! ct.Email }"/>
            <apex:column value="{! ct.Account.Name }"/>
        </apex:pageBlockTable>
    </apex:pageBlock>
</apex:page>

Add List View Filtering

Use {! listViewOptions } to get a list of list view filters available for an object. Use {! filterId } to set the list view filter to use for a standard list controller’s results.

Example of Standard List Controller


<apex:page standardController="Account" recordSetVar="accounts">
    <apex:form >    
    <apex:pageBlock title="Account List">
        <apex:repeat value="{!accounts}" var="a">
            <li> <apex:outputLink value="/{!a.Id}" /> </li>
        </apex:repeat>
    </apex:pageBlock>
    </apex:form>
</apex:page>
Using record Id at the root takes to the details page for that record.

Static Resources

First add a static resource, say jQuery to the organization from Setup | Custom Code | Static Resources.

Adding zipped static files to Visualforce Pages

Use the $Resource global variable and the URLFOR() function to reference items within a zipped static resource.

The URLFOR() function can combine a reference to a zipped static resource and a relative path to an item within it to create a URL that can be used with Visualforce components that reference static assets.

For example, {!URLFOR($Resource.vfimagetest, 'cats/kitten1.jpg')} returns a URL to a specific graphic asset within the zipped static resource, which can be used by the <apex:image> component.


<apex:page >
    <apex:image url="{!URLFOR($Resource.vfimagetest, 'cats/kitten1.jpg')}"></apex:image>
</apex:page>

Custom Controller

Custom controllers contain custom logic and data manipulation that can be used by a Visualforce page.

When using a custom controller a standard controller cannot be used.

The controller attribute of <apex:page> is use to define a custom controller to use with that page.

Create a Custom Controller

A custom controller is just an apex class. There are no interfaces that need to be implemented by the custom controller nor there needs to be class that the custom controller must extend.

Define methods with a getSomeExpression() will tell visualforce that this is a getter method and will use it accordingly.

Ex. getContacts() that returns List<Contact> being defined in the controller for a page, the using {! contacts } will tell visualforce page to use the records that the method got to update the page accordingly.

There are corresponding setter methods that let you submit values from the page back up to your controller.

An alternative to getters and setters is to use Apex properties. Properties are kind of a combination of a variable with getter and setter methods, with a syntax that groups them together more clearly. A simple property that references a custom object might be declared like this.

public MyObject__c myVariable { get; set; }.

Properties can be public or private, and can be read-only, or even write-only, by omitting the get or set.

Custom Controller Example


public class NewCaseListController {  
    public List<Case> getNewCases() {
        return [SELECT Id, CaseNumber, Status
               FROM Case
               WHERE Status='New'];
    }
}


<apex:page controller="NewCaseListController" >
    <apex:pageBlock title="New Cases List" id="cases_list">
    <li>
        <apex:repeat  value="{!NewCases}" var="Case" rendered="true" > 
        <p><apex:outputLink value="/{!Case.ID}">{!Case.CaseNumber}</apex:outputLink></p>
        </apex:repeat>
    </li>
    </apex:pageBlock>
</apex:page>
